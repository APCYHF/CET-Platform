# 项目名称：英语四六级服务平台（CET Prep Platform）

## 技术栈要求
- **前端**：Vue 3 + Vite + Vue Router + Pinia + Axios，UI 库建议使用 Element Plus（管理端）和 Vant（用户移动端/PC响应式）。
- **后端**：Spring Boot 2.7+ + MyBatis + MySQL 8.0，安全框架 Spring Security + JWT，接口文档自动生成使用 Knife4j。
- **AI 集成**：OpenAI API 或国内大模型（如智谱、通义千问），用于生词智能提取、作文基础批改。
- **文件存储**：本地存储或阿里云OSS（用于拍照上传图片）。
- **支付/人工批改**：先设计订单和状态机，人工精批后续对接（可预留接口）。

## 核心功能需求

### 1. 用户模块
- **登录/注册**：账号+密码，JWT 认证，支持记住密码。
- **个人中心**：头像、昵称、绑定学校/年级、学习目标（分数）、学习时长统计、我的生词本、我的错题本、我的订单/批改记录。

### 2. 四六级成绩查询
- 用户输入姓名、准考证号，调用第三方接口（模拟或预留对接官方渠道），展示总分及各单项分。
- 若无接口，提供手动录入成绩功能，并存入“历史成绩”折线图展示进步趋势。

### 3. 备考刷题功能（核心）

#### 3.1 模块模拟练习
分为听力、阅读、写作、翻译四大模块，每个模块内部可选题型（如听力含短篇新闻、长对话、篇章等）。

**听力模块要求**：
- 音频分段播放（按题干自然段或题目切分）。
- 倍速播放（0.8x、1.0x、1.2x、1.5x）。
- 实时显示听力原文（高亮当前播放句子），关键词标注（用户可打开/关闭）。
- 播放中可随时暂停并选择当前段落答案（单选/多选/填空）。
- 听完后统一提交，自动评分并给出解析。

**阅读模块要求**：
- 前端记录开始做题时间，用户完成每道题后暂存答案。
- 提交后系统自动批改，展示正确/错误答案及详细解析（长难句分析、选项翻译）。
- 支持错题自动加入“错题本”，用户可查看错题统计。

**写作 & 翻译模块**：
- 提供常见话题的模板、高级句型、素材库（按主题分类）。
- 拍照上传功能（OCR 识别或仅上传图片存为记录）。
- **免费 AI 基础批改**：调用大模型API，返回语法、拼写、连贯性评分及修改建议。
- **付费人工外教精批**：用户提交后生成订单（模拟支付），管理员后台指派外教批改，外教可上传批注后的文档/图片，用户端查看批改结果。

#### 3.2 生词本 & AI 智能建议
- **全局生词本**：任何页面遇到的单词（听力原文、阅读文章、题目选项、解析）都可长按/右键加入生词本。
- **做题后智能推荐**：
  - 管理端导入试题（文章、听力稿、题目）时，调用大模型分析提取“核心词汇”和“解题关键性词汇”，预先存储在 `question_word_suggestion` 表中。
  - 用户完成该套练习后，页面展示“本次练习建议掌握的生词”列表（来源于预存储的词汇），用户可一键勾选加入生词本。
  - 根据用户已选的词汇，后续推荐相同词根的题目或个性化复习计划。

#### 3.3 整套真题模拟
- 按考试标准组合题目（听力、阅读、翻译、写作），每部分限制建议时间（前端倒计时）。
- 只提供题目界面，用户按顺序作答，时间到自动锁止。
- 全部提交后：
  - 自动批改客观题（听力、阅读），主观题（写作、翻译）调用AI 初步评分，并给出参考分数。
  - 生成预估总分（转换算法参照四六级标准分）。
  - 按模块给出错题分析报告（正确率、用时、高频错误点），并推荐相关薄弱项练习题。

### 4. 背单词功能
- **词库来源**：
  - 四六级核心词 + 高频必考词（预置约3000词，包含词根、例句、音频）。
  - 我的错题本/生词本自动归入“重点词笔记本”。
- **学习计划**：
  - 基于艾宾浩斯遗忘曲线（1、2、4、7、15天复习点），用户可自定义每日新词量（10~50）。
  - 每日学习界面：卡片式展示单词、释义、例句，支持“认识/模糊/不认识”三种状态，系统自动调整复习间隔。
  - 提供听音选义、拼写测试、填空等练习模式。
- **进度追踪**：展示已学单词数、掌握率、预计完成日期。

### 5. 资料商城
- 商品类型：实体书（真题卷、词汇书）、电子资料（PDF课件、押题卷）、付费课程（录播课）。
- 用户可使用积分（通过每日签到、刷题、背单词获得）兑换，或实际支付（模拟微信/支付宝支付接口）。
- 后台管理商品上下架、库存、虚拟销量。

### 6. 口语备考
- 提供历年口语真题话题、模拟对话场景。
- 用户上传口语录音（3分钟内），调用免费语音识别 + 大模型给出流畅度、发音、语法反馈。
- 可预约人工1v1模拟考（付费功能）。

### 7. 后台管理系统
- **用户管理**：增删改查、封号、重置密码。
- **题库管理**：
  - 导入试题（Excel/文本），支持听力音频上传、图片解析。
  - 调用大模型自动提取核心词汇并存入建议表（可批量操作）。
  - 设置真题/模拟题的“建议用时”、难度系数、分值。
- **订单管理**：人工精批订单、资料商城订单。
- **批改任务分配**：管理员分配写作/翻译精批任务给外教，外教上传批改结果。
- **数据看板**：日活、刷题正确率分布、单词掌握情况统计。

## 数据库核心表设计（简要）
- `user`：用户基础信息、JWT token版本
- `word`：单词表（词根、释义、等级、音频URL）
- `user_vocabulary`：用户生词本（关联word，复习状态、下次复习时间）
- `question_bank`：题目表（题型、题干、选项、答案、解析、听力原文、音频URL）
- `exam_paper`：试卷表（真题/模拟，各部分时间限制）
- `paper_question`：试卷-题目关联表
- `user_answer_record`：用户做题记录（答案、耗时、是否正确）
- `word_suggestion`：题目/文章关联的建议词汇表（question_id / article_id, word_id, 权重）
- `composition_translation`：作文翻译题提交记录（图片URL、文本、AI批改结果、人工批改状态）
- `order`：商城订单、人工精批订单

## 非功能性要求
- 前端需适配手机H5和PC端（响应式布局）。
- 后端API遵循RESTful风格，统一返回 `{code, msg, data}`。
- 使用 Redis 缓存热点数据（如词库、试卷列表）。
- 大模型调用需异步处理（管理端导入试题时使用消息队列，避免阻塞）。
- 所有敏感接口（修改密码、支付）需额外验证。
